ngOnChanges

It is specifically designed to handle changes to the component's input properties.

  • normally as per create a parent to child components use of input decorator
  • In child component add these given below for ngOnChanges
  • Import the OnChanges, SimpleChanges on child components
  • add OnChanges on export class after OnInit
  • its only working for string, not for Arrays and Objects

ngOnInit

  • Always it will called after ngOnchanges
  • in child components add these given below for ngOninit
  • use like onChanges Import ngOninit and add interface using comma(,)
  • write a function ngOninit "ngOninit()"
  • impartant!! it is mostly used for runs a data before a component should be created

ngDoCheck

  • it is used for check the changes for all types including strings, arrays, objects
  • ngOnChanges is only for get changes of strings but ngDoCheck is for all to check the changes

ngOnDestroy

  • it is used to destroy the component (which mean hide the entire component)
